# Grafana
> Grafana란, 데이터 시각화 및 모니터링 도구

| 구분         | 설명                                                                                |
| ---------- | --------------------------------------------------------------------------------- |
| 용도     | 서버, 애플리케이션, 네트워크 등에서 수집한 **메트릭(metric)** 데이터를 시각화                                 |
| 형태     | 웹 기반의 **대시보드(dashboard)** 형태로 차트, 그래프, 게이지 등 표시                                   |
| 데이터 소스 | Prometheus, InfluxDB, Elasticsearch, Loki, Azure Monitor, AWS CloudWatch 등과 연동 가능 |
| 언어/구조  | Go(백엔드) + React(프론트엔드)로 작성된 오픈소스                                                  |
| 운영 방식  | 직접 서버에 설치하거나 Docker, Kubernetes, 또는 Grafana Cloud로 운영 가능                          |

## 특징
- 메트릭(metrics), 로그(logs), 추적(traces) 등 다양한 데이터가 저장된 곳(데이터베이스, 클라우드 서비스 등)에 상관없이 쿼리, 시각화, 알림, 탐색을 할 수 있게 해주는 오픈소스 소프트웨어
- 데이터 통합 (Unify your data, not your database)
  - 사용자가 데이터를 특정 저장소로 이전(ingest)할 필요 없이, 이미 존재하는 여러 데이터 소스들을 연결해 한 화면(대시보드)에서 볼 수 있도록 함
- 데이터 누구나 볼 수 있음 (Data everyone can see)
  - 누구나 필요한 데이터를 쉽게 보고 활용할 수 있는 문화를 조성하고, 데이터 사일로(silo) 즉, 팀 간 정보 단절을 허물며 팀이 더 주도적으로 일할 수 있도록 도움
- 공유 가능한 대시보드 (Dashboards that anyone can use)
  - 사용자가 만든 대시보드를 팀원이나 조직 내 다른 사람들과 공유할 수 있고, 공동으로 데이터 가능
- 유연성과 다양성 (Flexibility and versatility)
  - 쿼리, 변환, 시각화 옵션 등이 풍부해서 여러 데이터 조건에 맞게 패널을 구성할 수 있고, 특정한 요구사항에 맞출 수 있음

## 핵심 기능
| 기능                                            | 설명                                                                                          |
| --------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **패널(Visualization Panels)**                  | 히트맵, 히스토그램, 그래프, 지도(geomap) 등 다양한 시각화 유형을 제공하여 데이터를 다양한 방식으로 표현 가능 |
| **알림(Alerting)**                              | 메트릭 조건을 설정해두면 임계치를 넘었을 때 이메일, Slack, PagerDuty 등 알림 가능 |
| **주석(Annotations)**                           | 그래프 위에 이벤트나 메타데이터를 표시해, 특정 시점의 이벤트를 시각적으로 표시 가능  |
| **대시보드 변수(Templating / Dashboard Variables)** | 변수(예: 서버 이름, 리전 등)를 사용해 하나의 대시보드를 여러 환경에 재사용 가능 |
| **탐색(Explore)**                               | 대시보드에 고정하지 않고 쿼리를 직접 실행하면서 데이터 탐색 가능. 메트릭과 로그 간 전환 지원 |
| **플러그인 시스템**                                  | 데이터 소스 연결 플러그인, 패널 플러그인 등이 있어 여러 시스템과 통합 가능 |
| **보안 및 인증**                                   | LDAP, OAuth 등 다양한 인증 방식 지원 (Enterprise 버전: 팀 기반 접근 제어(RBAC) 강화) |
| **스케일 & 확장성**                                 | Enterprise 버전: 대규모 데이터 볼륨과 고가용성 환경 지원   |
